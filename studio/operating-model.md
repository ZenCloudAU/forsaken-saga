# Studio Operating Model

## Core Structure

The Forsaken Saga operates as a small modern independent studio project.

Phil acts as Founder, Creative Director, Lead Designer, World Architect, and AI Architect.

The project copilot acts as Lead Developer support, Product Manager, UX Designer, Technical Producer, Archivist, QA Coordinator, Release Manager, and Documentation Lead.

## Operational Philosophy

The project is intentionally built in public.

The repository is not only code. It is:
- the workshop floor,
- the design archive,
- the learning journal,
- the lore bible,
- the sprint tracker,
- the technical notebook,
- the devlog,
- and the public production diary.

## Pace Control

The project uses sprint discipline to prevent uncontrolled expansion.

One active sprint at a time.

One playable milestone at a time.

One learning objective at a time.

Ideas are always captured.

Ideas are not automatically implemented.

## Build Rule

Every session should create a visible artefact:
- code,
- a note,
- a mechanic,
- a screenshot,
- a commit,
- a devlog,
- a design decision,
- a diagram,
- or a playable interaction.

## Primary Goal

Phil focuses primarily on learning coding and building mechanics.

The supporting system handles administration, organisation, documentation, roadmap management, architectural continuity, sprint tracking, and publishing support.
