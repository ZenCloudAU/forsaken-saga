# Command Log

This file tracks commands, scripts, installs, exports, builds, and technical actions used during development.

The purpose is educational continuity.

Phil should be able to review this file months later and understand:
- what commands were executed,
- why they were used,
- what they changed,
- and how the workflow evolved.
